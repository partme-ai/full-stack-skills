## 2. 菜单归位与整合规则

- **监控 & 沙盒**：不再作为一级菜单。`监控` 功能分散至 `概览` 与 `实例`；`沙盒`（WASM/Docker）整合进 `工具 Tools` 的执行环境管理。
- **日志审计**：功能拆分。`巡检与风险发现` 归入 `概览`；`操作审计与系统日志` 归入 `设置 Settings` 或 `概览` 的事件流。
- **自动化 & 记忆库 & 文档**：从原有的二级或深层菜单提拔为一级菜单，以突出 AI 原生能力。
- **安装向导 & 节点管理**：统一收敛至 `实例 Instances` 模块。

## 3. 详细功能分布

### 3.1 概览 Dashboard
- **平台总览**：全局资源、活跃实例、节点分布。
- **健康看板**：实例健康、节点健康、服务状态（原“监控”模块内容）。
- **风险告警**：巡检发现的风险、异常实例告警（原“日志审计”巡检部分）。
- **任务简报**：最近失败任务、最近审计事件流。

### 3.2 实例 Instances
- **实例概览**：所有引擎（OpenClaw / ZeroClaw / OctoClaw）的列表与运行状态。
- **创建与部署**：
  - 安装向导：环境预检、安装依赖、配置生成、服务安装、启动验证。
  - 批量部署：多目标主机的一键下发。
- **运维控制**：升级、回滚、重启、日志查看、性能监控。
- **节点管理**：远程主机管理、执行器（Node Agent）安装与状态。

### 3.3 智能体 Agents Management
- **Agent 市场**：官方及社区提供的智能体角色模板。
- **我的智能体**：已配置的 Agent 列表。
- **套装模板**：多 Agent 协作套装（Swarm / Team）。
- **生命周期**：版本管理、分发至实例、升级/回滚策略。

### 3.4 模型 Models
- **全部模型**：统一的模型接入列表。
- **凭证中心**：API Key 管理、多平台厂商接入。
- **模型订阅**：官方/聚合平台接入模板、配额监控、注册引导。
- **模型代理**：统一网关、服务端代理、故障转移（Failover）、熔断恢复。
- **路由策略**：基于成本/速度/能力的自动路由。

### 3.5 技能 Skills
- **Skills 市场**：可供 Agent 调用的技能（Function Calling）商店。
- **已安装技能**：技能的分类管理、启停控制。
- **开发与调试**：技能在线测试、权限范围（Scope）定义。

### 3.6 工具 Tools
- **MCP 市场**：Model Context Protocol 兼容插件商店。
- **连接器 Connectors**：渠道（WeCom, Slack, Telegram 等）接入管理。
- **执行环境**：WASM 沙盒、Docker 隔离环境、OpenSandbox 配置。
- **工具链集成**：外部 API 与 SDK 的连接配置。

### 3.7 自动化 Automation
- **工作流编排**：可视化或代码化的任务流水线。
- **定时任务**：Cron 任务、周期性巡检触发。
- **事件触发**：基于 Webhook 或系统事件的自动化响应。
- **执行审计**：自动化任务的历史记录与成功率分析。

### 3.8 记忆库 Memory Management
- **记忆后端**：向量数据库（Vector DB）、图数据库（Neo4j）管理。
- **知识索引**：文档上传、网页抓取、RAG 索引构建。
- **记忆策略**：Recall 召回机制、记忆衰减与压缩、跨 Agent 共享。
- **数据管理**：记忆片段查看、手动干预与清理。

### 3.9 设置 Settings
- **平台管理**：RBAC 权限、安全设置、更新与版本、系统备份。
- **审计日志**：系统日志、操作审计、异常追踪（原“日志审计”核心）。
- **存储与目录**：安装路径、数据卷、缓存清理。

### 3.10 文档 Docs
- **知识中心**：安装手册、接入文档、模型能力矩阵。
- **帮助与支持**：FAQ、故障排查（Troubleshooting）、社区链接。
- **API 引用**：控制面 API 定义与 Swagger 文档。
