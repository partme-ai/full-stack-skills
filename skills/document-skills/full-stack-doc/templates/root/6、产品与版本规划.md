## 1. 产品定位

`{Name} = 多引擎智能体控制平面。`

一句话：

`面向 OpenClaw / ZeroClaw / OctoClaw 的安装、纳管、配置、运维、扩展与审计平台。`

## 2. 产品骨架来源

### 2.1 从 1Panel 借什么

- 借信息架构，不只借皮肤
- 左侧一级导航 + 顶部状态栏 + 首页总览 + 列表页 + 详情页 + 安装向导
- 一键安装脚本 + CLI 工具 + Web 面板 + 节点/主机管理 + 应用市场
- 资源对象化：主机、应用、任务、备份、日志、商店都做成标准对象

### 2.2 从 clawpanel 借什么

- 安装 / 诊断 / 修复主链
- AI 助手做“受控运维代理”而不是闲聊机器人
- Web 控制面 + 同机执行代理分层
- 多实例与远程实例代理思路

## 3. {Name} 的差异化

{Name} 不应只是 1Panel 或 clawpanel 的换壳版，而要强化：

- 多引擎统一适配
- 游戏化 6 步安装向导
- 多实例批量纳管
- 渠道 / Skill / MCP 卡片式安装
- 模型订阅与代理平台目录
- 受控 AI 安装助手
- 审计、回滚、任务阶段结果
- Web-only 控制台 + Go Core + Node Agent

## 4. 形态策略

### 4.1 {Name} Web

- 唯一控制台形态
- 面向服务器部署与远程访问
- 负责登录、实例管理、模型订阅、渠道配置、技能 / 工具治理、监控、日志审计、任务回查

### 4.2 {Name} Core

- `Go` 服务
- 负责 API、任务编排、状态机、审计、权限、节点 / 实例注册、批量任务
- 充当控制面与执行面的统一编排核心

### 4.3 {Name} Node Agent

- `Node.js + TypeScript` 执行代理
- 部署在被管机器上，跟引擎同机
- 负责安装、预检、配置生成、服务启停、日志采集、健康检查、回滚

## 5. 版本层级

| 层级 | 名称 | 目标 |
|------|------|------|
| `P0` | 安装闭环 | 三引擎安装、健康检查、纳管、回滚 |
| `P1` | 控制面骨架 | 首页、实例、模型、监控、设置 |
| `P2` | 平台治理 | 沙盒、智能体、技能商店、工具商店、模型订阅 |
| `P3` | 平台增强 | 日志审计、批量运维、平台扩展与高级协作 |

## 6. 一级导航冻结

{Name} 一级导航固定为：

1. 首页
2. 实例
3. 模型
4. 监控
5. 沙盒
6. 智能体
7. 技能商店
8. 工具商店
9. 日志审计
10. 设置

说明：

- `实例` 承接安装向导、节点与远程主机、部署 / 升级 / 回滚 / 批量任务
- `模型` 承接模型提供商、凭证、路由、模型代理与模型订阅目录，不拆分额外一级菜单
- `日志审计` 承接巡检总览、操作审计、异常日志、实时事件流和任务阶段日志
