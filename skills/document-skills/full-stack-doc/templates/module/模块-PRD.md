     - 收入说明文案：固定展示文案等运营那边确认后定（待定）
     - 要求到达：默认比客户预计的上门时间早10分钟
     - 客户期望到达：取`work_order_info.appointment_date + appointment_time`，展示用户下单时预定的服务开始–结束时间
     - 阶段时间：抢单、到达、服务、完成，这几个时间以师傅实际点击提交为准，前端统一展示为"MM-dd HH:mm"格式，例如：12-29 15:02。未完成的阶段显示"--"

2. **订单操作按钮规则**（根据订单状态显示不同按钮）：
   - **待接单状态**：
     - 【取消订单】按钮：点击需二次确认弹窗，确认后订单状态更新为已取消并从当前列表移除
     - 【联系客户】按钮：点击直接调用系统拨号
     - 【指派师傅】按钮：点击按钮进入指派弹窗，选择可用师傅后更新接单信息
   - **待转派状态**：
     - 【转派订单】按钮：点击按钮触发师傅弹窗，完成后当前师傅信息更新为新师傅，并记录转派时间
   - **待上门状态**：
     - 底部整条按钮展示为「X分钟后开始」（倒计时），或「已到预约时间，尽快上门」（已超时）
     - 按钮为禁用态，仅做提示，无点击事件
   - **服务中状态**：
     - 展示距离服务结束的倒计时，实时按本地时间刷新
     - 显示"剩余X分钟"或"已超时X分钟"

3. **倒计时功能规则**：
   - **待上门倒计时**：
     - 当前时间 < expected_arrive_time：底部整条按钮展示为「X分钟后开始」，用分钟倒计时，实时刷新
     - 当当前时间 ≥ expected_arrive_time 且 service_start_time 为空：底部按钮文案切换为「已到预约时间，尽快上门」，不再显示倒计时
     - 进入详情页时按当前时间重新计算剩余时间；若停留在该页面，文案每分钟自动更新一次
   - **服务中倒计时**：
     - 根据服务时长字段（单位：分钟，如120分钟）计算本次服务计划结束时间：计划结束时间 = 服务开始时间 + 服务时长
     - 当前时间 < 计划结束时间时，展示"剩余X分钟"
     - 当前时间 ≥ 计划结束时间且师傅未点击"结束服务"时，展示"已超时X分钟"
   - **倒计时计算规则**：
     - 不依赖`setInterval`递减，而是依赖**系统时间差**
     - 在`onShow`时强制重算，防止后台挂起导致计时偏差

**数据来源**：
- **接口**：`GET /workorder/workOrder/{orderId}`
- **数据表**：
  - `work_order_info`（工单主表）
  - `work_order_item`（工单明细表）
  - `worker_user`（师傅表）
- **字段映射**：
  - 订单编号：`work_order_info.order_no`
  - 订单收入：`work_order_info.payment_price`（需计算固定收入+上门费）
  - 要求到达：`work_order_info.expected_arrive_time`（如果不存在，需计算：appointment_time - 10分钟）
  - 客户期望到达：`work_order_info.appointment_date + appointment_time`
  - 阶段时间：`work_order_info.accept_time`（抢单）、`work_order_info.arrive_time`（到达）、`work_order_info.service_start_time`（服务）、`work_order_info.service_end_time`（完成）
  - 师傅姓名：`worker_user.name`
  - 师傅电话：`worker_user.phone_mob`（注意：不是`phone`字段）
  - 师傅等级：`worker_user.level_id`（需要转换为等级名称）

**异常处理**：
- **订单不存在**：提示"订单不存在"，返回上一页
- **订单数据加载失败**：显示"加载失败，请重试"提示，支持重试
- **倒计时计算异常**：显示"--"，不影响其他功能

## 2. 产品概述 (Product Overview)

### 2.1 功能定位

设备中心是 {Name} 的核心入口，提供多 OpenClaw 实例的注册、分组、监控、配置管理与单实例管理（点击某实例后进入的完整管理面板），通过游戏化向导与可视化表单提供与 OpenClaw 等同的配置能力，并支持基线/模板/一致性检测与批量下发。

## 4. 功能需求 (Functional Requirements)

### 5.1 实例列表（DC-01）

**功能ID**：DC-01  
**功能名称**：实例列表  

**功能描述**：设备中心默认页。展示已注册 OpenClaw 实例的卡片或列表视图，含在线状态、实例名、一致性状态、版本、标签、Agent 数、会话数、配置差异项数；支持添加实例、筛选、视图切换、批量操作；操作包括管理 →、查看差异、从基线恢复、下发基线、关联基线（按状态显示）。与《{Name} 需求分析文档 V1》§5.1.1 一致。

**用户故事**：作为运维人员，我希望在设备中心看到所有实例的状态与一致性，以便快速进入管理或执行下发/恢复。

**功能规则**：

1. **显示规则**：卡片视图栅格 3～4 列，卡片内在线状态点（8px）+ 实例名 + 一致性状态 ●（已同步/已修改/待下发/下发失败/未纳管，色与 DNA 一致）、版本、标签、Agent 数/会话数/配置差异项数、勾选框与操作按钮。列表视图为表格，列与卡片信息对应，操作列右对齐。筛选区：分组 ▾、标签 ▾、在线状态 ▾、搜索框（占位「搜索实例名...」）；视图切换「卡片 | 列表」。
2. **业务规则**：一致性状态与差异项数由「当前实例配置 vs 基线」计算（通过 WS config.get 拉取实例配置）；未纳管实例可关联基线后纳入管理。批量操作：批量启停、批量重启、批量更新插件等。
3. **交互规则**：切换视图保留筛选条件。勾选多条后显示批量操作栏；选择操作并确认后请求执行。点击「管理 →」路由到该实例单实例管理，默认 Tab「概览」。从基线恢复、同步到基线需二次确认（与总纲 §2.4 危险操作一致）。

**数据来源**：

- 实例列表：管理后台 {Name} 数据库（instance、instance_group、instance_tag）；在线状态、会话数、Agent 数等通过各实例 **Gateway WebSocket** `health`、`status` 及统计能力获取；一致性状态与差异项数由聚合层对每个实例 **WS config.get** 与基线 diff 计算。

**异常处理**：

- **网络异常**：列表加载失败展示 Toast 或横幅，支持重试。单实例离线时该实例显示离线态，不影响其他实例展示。
- **无实例**：空状态展示插画+「立即添加实例」引导（与总纲 §2.3 一致）。
- **从基线恢复/同步失败**：模态展示错误原因，支持重试。

**验收标准**：

1. ✅ 筛选（分组/标签/在线状态/搜索）正确过滤实例列表。
2. ✅ 卡片/列表视图切换正确，切换后保留筛选条件。
3. ✅ 点击「管理 →」进入该实例单实例管理，默认 Tab「概览」。
4. ✅ 按一致性状态正确显示「查看差异」「从基线恢复」「下发基线」「关联基线」等操作。
5. ✅ 从基线恢复、同步到基线有二次确认，执行后状态与基线一致。
6. ✅ 批量勾选后批量操作栏出现，执行后结果正确反馈。

### 5.2 实例分组（DC-04）

**功能ID**：DC-04  
**功能名称**：实例分组  

**功能描述**：按业务线/地域/环境等维度对实例进行分组与打标签；分组/标签 CRUD、与实例多对多关联；实例列表筛选依赖分组与标签。界面与交互遵循 [{Name} 视觉与交互 DNA 规范](../../9、{Name}-视觉与交互DNA规范.md) §5.3 列表与表格、§5.2 表格与卡片。

**用户故事**：作为运维人员，我希望将实例按业务线或地域分组并打标签，以便在实例列表与下发给定时快速筛选。

**功能规则**：

1. **显示规则**：独立页「设备中心 > 实例分组」；分组与标签可分 Tab 或左右区；表格或卡片：分组名/标签名、关联实例数、操作（编辑、删除、管理实例）；主按钮「+ 新建分组」「+ 新建标签」；空状态插画+引导（DNA §5.6）。
2. **业务规则**：分组与标签存 {Name} 数据库；实例与分组多对多；删除分组前若有实例关联需提示或先解绑。
3. **交互规则**：新建/编辑 → 表单（名称、描述等）→ 保存后列表刷新；管理实例 → 多选实例关联到该分组。

**数据来源**：{Name} 数据库 instance_group、instance_tag、instance_group_relation；实例列表筛选时带 group_id/tag_id 参数。

**异常处理**：保存失败 Toast；删除有关联实例的分组时提示「请先解除实例关联」或二次确认。

**验收标准**：

1. ✅ 分组/标签 CRUD 正常，列表正确展示关联实例数。
2. ✅ 管理实例可多选实例关联到分组；实例列表筛选按分组/标签正确过滤。
3. ✅ 删除有关联的分组有提示或二次确认。

### 5.3 集群概览（DC-05）

**功能ID**：DC-05  
**功能名称**：集群概览  

**功能描述**：多实例聚合指标页：健康状态（正常/异常/离线占比）、总会话量、总用量、渠道状态；支持时间筛选与刷新；可选下钻到实例列表。状态色与 [DNA 规范](../../9、{Name}-视觉与交互DNA规范.md) §3.1 功能色、§3.1.2 一致性状态色 一致。

**用户故事**：作为运维人员，我希望在一页看到所有实例的聚合健康与用量，以便快速发现异常。

**功能规则**：

1. **显示规则**：独立页「设备中心 > 集群概览」；顶栏时间范围下拉、刷新按钮；指标卡片行（健康、会话量、用量、渠道状态）；可选饼图/柱状图，图例 12px。
2. **业务规则**：数据由聚合层汇总各实例 **Gateway WebSocket** health、status 及统计接口。
3. **交互规则**：时间筛选/刷新 → 重新请求聚合数据；下钻（若有）→ 跳转实例列表并带筛选。

**数据来源**：各实例 WS health、status；聚合层或 {Name} 统计表。

**异常处理**：加载失败 Toast/横幅、重试；部分实例离线时聚合仍展示，离线数计入「异常/离线」占比。

**验收标准**：

1. ✅ 指标卡片与可选图表正确展示；时间筛选与刷新生效。
2. ✅ 下钻（若有）跳转实例列表并带正确筛选参数。

### 5.4 配置管理 — 配置模板（DC-02e）

**功能ID**：DC-02e  
**功能名称**：配置模板  

**功能描述**：可复用的配置片段（Agent/渠道/插件等），分类管理、版本管理；从模板应用至基线；列表与新建/编辑表单。遵循 [DNA 规范](../../9、{Name}-视觉与交互DNA规范.md) §5.4 表单与配置、§5.2 表格。

**用户故事**：作为运维人员，我希望维护一批配置模板并可从模板应用到基线，以便快速生成标准配置。

**功能规则**：

1. **显示规则**：独立子页「配置管理 > 配置模板」；表格 模板名、分类、版本、操作；主按钮「+ 新建模板」；新增/编辑弹窗或抽屉（模板名、分类、内容 JSON 或表单）。
2. **业务规则**：模板存 {Name} 数据库；分类可选 Agent/渠道/插件等；从模板应用在基线配置页操作，选择模板后合并到当前基线（同 DC-02a 乐观锁）。
3. **交互规则**：新建/编辑 → 保存后列表刷新；从模板应用入口可在基线配置页。

**数据来源**：{Name} 数据库 config_template；从模板应用写基线表。

**异常处理**：保存失败 Toast；从模板应用时若基线版本冲突同 DC-02a。

**验收标准**：

1. ✅ 模板 CRUD 正常；从模板应用至基线后基线内容正确合并。
2. ✅ 分类与版本展示正确。

### 5.5 配置管理 — 基线配置（DC-02a）

**功能ID**：DC-02a  
**功能名称**：基线配置  

**功能描述**：可视化表单编辑「基线配置」，对应 OpenClaw 配置结构，分 7 大分类（基础/认证/Agent/渠道/插件/监控/会话等）；支持版本管理（版本号、备注）、导出 JSON、从模板应用。

**用户故事**：作为运维人员，我希望在管理后台编辑基线配置并保存版本，以便统一下发给多实例且可追溯。

**功能规则**：

1. **显示规则**：7 大分类以 Tab 或左侧导航组织；表单字段与 openclaw.json 结构对齐；版本号与备注在页顶或版本历史区展示。
2. **业务规则**：必填与格式校验（失焦校验，与总纲 §2.4 一致）；保存后生成新版本、版本递增。**基线版本冲突处理**：保存时传当前 version，后端乐观锁校验；若后端 version > 前端，返回冲突错误，前端提示「配置已被他人更新，请刷新后重新编辑」并拦截保存。
3. **交互规则**：从模板应用 → 选择模板后合并到当前基线。导出 JSON → 下载文件可被 OpenClaw 加载。

**数据来源**：

- 基线存 {Name} 数据库（baseline_version 等）；读写为管理后台自有 API；与实例交互通过 **Gateway WebSocket** config.set/config.apply 在下发时使用。

**异常处理**：

- **冲突**：保存时若版本冲突，展示上述提示，不提交；用户刷新后重新编辑再保存。
- **校验失败**：字段下方红色错误文案，不提交。
- **网络异常**：保存失败 Toast 或模态，支持重试。

**验收标准**：

1. ✅ 编辑保存成功，版本历史可查。
2. ✅ 导出 JSON 可被 OpenClaw 加载。
3. ✅ 多人同时编辑时，后保存者遇到版本冲突则提示「配置已被他人更新，请刷新后重新编辑」并拦截保存。
4. ✅ 从模板应用后基线内容正确合并。

### 5.6 配置管理 — 一致性总览（DC-02b）

**功能ID**：DC-02b  
**功能名称**：一致性总览  

**功能描述**：表格展示「实例 × 一致性状态」及差异项数；操作：查看差异、下发基线、从基线恢复、关联基线（未纳管）；支持按状态/分组筛选；底部「全部下发」「仅下发待下发」「按分组下发」「按标签下发」。

**用户故事**：作为运维人员，我希望在一页看到所有实例与基线的一致性及差异项数，以便批量下发或恢复。

**功能规则**：

1. **显示规则**：表格列 实例、状态、差异项数、操作；状态列 ● + 已同步/已修改/待下发/下发失败/未纳管（色与 DNA 一致）；操作列按状态显示查看差异、下发基线、从基线恢复、关联基线等。
2. **业务规则**：状态与差异项数由「当前实例配置 vs 基线」计算（对各实例 **WS config.get** 与基线 diff）；未纳管实例可关联基线后纳入管理。
3. **交互规则**：查看差异 → 打开 diff 弹窗或跳转单实例配置页；下发基线/从基线恢复 → 可多选实例后执行；从基线恢复需二次确认。

**数据来源**：

- 实例列表与基线来自 {Name} 数据库；每个实例当前配置通过 **Gateway WebSocket** `config.get` 拉取；一致性由聚合层计算后返回。

**异常处理**：

- **部分实例离线**：该实例状态可为「未纳管」或标记离线，不影响其他行操作。
- **下发/恢复失败**：单实例失败时记录原因，可重试；整批失败时模态展示。

**验收标准**：

1. ✅ 状态与差异项数与实际「实例 config vs 基线」一致。
2. ✅ 查看差异内容与单实例配置页 diff 一致。
3. ✅ 下发与从基线恢复流程可用，结果与下发记录一致。
4. ✅ 未纳管实例关联基线后纳入管理并出现在列表中。

### 5.7 配置管理 — 下发任务（DC-02c）

**功能ID**：DC-02c  
**功能名称**：下发任务  

**功能描述**：新建下发 → 步骤 1 选来源（基线/模板）→ 步骤 2 选目标实例（可全选、按分组）→ 步骤 3 预览每实例 diff → 步骤 4 确认下发并展示每实例结果（成功/失败+原因）。**操作人必填**并写入下发记录（总纲 §5.4）。**离线实例策略**：模式 A 立即失败；模式 B 自动重试最多 3 次；等待重试可查看与取消。

**用户故事**：作为运维人员，我希望通过四步向导将基线或模板下发给多台实例，并看到每台成功/失败结果，以便审计与重试。

**功能规则**：

1. **显示规则**：四步步骤条；每步内容区最大宽度约 720px；步骤 3 预览 diff 等宽字体，删除线+红表示旧值、下划线+绿表示新值；步骤 4 每实例一行 ✅/❌ 与重试按钮。
2. **业务规则**：下发时对每个目标实例通过 **Gateway WebSocket** `config.set`/`config.apply` 写入配置并触发热重载。失败记录原因并支持单实例重试。**离线实例**：模式 A 则该实例立即失败并记录；模式 B 则进入重试队列，最多 3 次，等待重试状态可查看与取消。**操作人**：当前登录用户写入 deployment_task，与总纲 §5.4 审计一致。
3. **交互规则**：每步可上一步/下一步；步骤 4 确认后异步或同步执行，展示每实例结果；重试仅针对失败实例。

**数据来源**：

- 来源：基线/模板来自 {Name} 数据库。目标实例列表来自 {Name} 数据库。执行：Infrastructure 层按实例建立 **Gateway WebSocket** 连接，调用 config.set/config.apply；结果写回 deployment_task、deployment_task_result。

**异常处理**：

- **部分实例失败**：该实例行显示 ❌ 与原因，可点击重试；整任务不回滚已成功实例。
- **全部失败**：步骤 4 展示全部 ❌，可全部重试或返回修改来源/目标。
- **网络异常**：请求超时或断开按失败处理，记录原因。

**验收标准**：

1. ✅ 四步流程完整，步骤 3 预览 diff 与步骤 4 结果一致。
2. ✅ 下发记录可查，且含操作人、时间、目标实例、每实例结果。
3. ✅ 离线实例：模式 A 立即失败并记录；模式 B 进入重试队列，可查看与取消，最多重试 3 次。
4. ✅ 单实例失败可重试，重试后结果更新。

### 5.8 配置管理 — 下发记录（DC-02d）

**功能ID**：DC-02d  
**功能名称**：下发记录  

**功能描述**：历史下发任务列表：时间、**操作人**、配置来源、目标实例、变更摘要、每实例结果；支持筛选与查看详情。与总纲 §5.4 审计日志一致。

**用户故事**：作为运维人员，我希望查看历史下发记录及操作人，以便审计与排查问题。

**功能规则**：

1. **显示规则**：表格列 时间、操作人、配置来源、目标实例、变更摘要、每实例结果（可展开或详情页）；支持按时间、操作人、来源筛选。
2. **业务规则**：所有下发与基线恢复/同步操作写入 operation_log 或 deployment_task，**操作人必填**；保存期限与总纲 §5.4 一致（≥90 天）。
3. **交互规则**：点击某条可查看详情（每实例成功/失败及原因）。

**数据来源**：

- deployment_task、deployment_task_result 及操作人来自 {Name} 数据库；只读展示。

**异常处理**：

- **无记录**：空状态展示「暂无下发记录」类文案。

**验收标准**：

1. ✅ 记录完整可查，每条含操作人、时间、配置来源、目标实例、结果。
2. ✅ 与下发任务执行结果一致；筛选与详情正确。

### 5.9 单实例管理 — 配置管理（DC-03）

**功能ID**：DC-03  
**功能名称**：单实例配置管理  

**功能描述**：单实例内「配置管理」页：顶部展示与基线差异 N 项及状态；按钮「查看差异」「从基线恢复」「同步到基线」；下方为 7 大分类表单；与基线不同的字段有「⚠ 差异」标记；支持保存、重置、与基线对比、导出 JSON。

> **模型服务关联**：单实例配置中的「模型配置」节点支持从**模型服务模板库**（[4.6](../6、模型服务/{Name}-模型管理-PRD-V1.md) MM-07）应用模板；Key 池中的 API Key 可在下发时注入实例配置（加密或引用方式）。详见 《4.6 §2.6.3》

**用户故事**：作为运维人员，我希望在单实例配置页直接编辑并保存，或从基线恢复/同步到基线，以便单实例与基线一致或保留差异。

**功能规则**：

1. **显示规则**：顶部横幅「与基线差异 N 项」+ 三按钮；左侧配置分类（或 Tab）；右侧 7 大分类表单；差异字段 ⚠ 标记；底部保存、重置、与基线对比、导出 JSON。
2. **业务规则**：保存后通过该实例 **Gateway WebSocket** `config.set` 写回并触发热重载。从基线恢复（用基线覆盖当前实例）、同步到基线（用当前实例覆盖基线）需**二次确认**（与总纲 §2.4 危险操作一致）。
3. **交互规则**：查看差异 → 打开 diff 视图；保存 → 提交当前表单，成功 Toast；恢复/同步 → 确认框 → 执行后刷新状态。

**数据来源**：

- 当前实例配置：**Gateway WebSocket** `config.get`。基线：{Name} 数据库。保存：**WS config.set** 写回该实例。

**异常处理**：

- **保存失败**：模态展示错误原因（如实例离线、网络超时），支持重试。
- **从基线恢复/同步失败**：同上。

**验收标准**：

1. ✅ 差异 N 项与 diff 内容正确。
2. ✅ 保存后实例配置更新且热重载生效。
3. ✅ 从基线恢复后实例与基线一致；同步到基线后基线更新。
4. ✅ 危险操作均有二次确认。
